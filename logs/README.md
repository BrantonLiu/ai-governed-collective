# 工作日志目录

## 📋 说明

这个目录记录了项目的日常工作日志。

## 📁 结构

```
logs/
├── 2026-02/           # 2026年2月
│   ├── 2026-02-26.md
│   ├── 2026-02-27.md
│   └── ...
├── 2026-03/           # 2026年3月
│   └── ...
└── README.md          # 本文档
```

## 📝 如何写日志

1. 进入对应月份目录
2. 创建当天的文件 `YYYY-MM-DD.md`
3. 按照 [模板](../templates/LOG-TEMPLATE.md) 记录工作
4. 提交到Git仓库

## 🎯 日志的作用

- **个人**: 复盘和成长
- **团队**: 了解彼此工作
- **项目**: 保留决策过程
- **AI**: 理解上下文

## 💡 写作建议

- 简洁但不遗漏关键信息
- 重点记录决策和思考过程
- 定期回顾和总结
