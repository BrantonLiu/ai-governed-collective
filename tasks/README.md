# 需求和任务目录

## 📋 说明

这个目录管理项目的所有需求和任务。

## 📁 结构

```
tasks/
├── open/              # 待承接的任务
│   └── TASK-001-xxx.md
├── in-progress/       # 进行中的任务
│   └── TASK-002-xxx.md
├── completed/         # 已完成的任务
│   └── TASK-003-xxx.md
└── README.md          # 本文档
```

## 🔄 任务生命周期

```
1. 创建任务 → open/
2. 承接任务 → 在任务文件中标注承接者
3. 移动任务 → in-progress/
4. 完成任务 → 填写完成信息
5. 移动任务 → completed/
```

## 📝 如何创建任务

1. 查看 `open/` 目录中现有任务，避免重复
2. 复制 [任务模板](../templates/TASK-TEMPLATE.md)
3. 命名为 `TASK-{编号}-{简短描述}.md`
   - 编号从001开始递增
   - 描述使用kebab-case
4. 填写任务信息
5. 放入 `open/` 目录
6. 提交到Git

## 📋 任务编号规范

- **001-099**: 核心功能
- **100-199**: 文档改进
- **200-299**: 工具开发
- **300-399**: 社区建设
- **400-499**: 运维支持

## ✅ 任务验收

任务完成后需要：
1. 填写实际工时
2. 记录遇到的问题和解决方案
3. 计算贡献分
4. 更新贡献记录
